1) Which of the following best distinguishes Continuous Delivery (CD) from Continuous Deployment?
A. Delivery includes manual testing; Deployment doesn’t
B. Delivery stops before production release; Deployment automates it
C. Deployment happens weekly; Delivery daily
D. They are the same
Answer: B

2) Which YAML (YAML Ain't Markup Language) file defines a GitHub Actions workflow?
A. .github/settings.yml
B. .github/workflows/ci.yml
C. .git/config
D. package.json
Answer: B

3) What is a key reason to use branch protections in Continuous Integration/Continuous Delivery (CI/CD)?
A. To make branching harder
B. To ensure only tested, reviewed code merges to main
C. To slow down releases
D. To prevent code reviews
Answer: B

4) What benefit does Infrastructure as Code (IaC) provide?
A. It hides infrastructure details
B. It enables consistent, versioned, reproducible environments
C. It eliminates the need for documentation
D. It allows only manual configuration
Answer: B

5) Which of the following is NOT typically part of a Continuous Integration (CI) pipeline?
A. Unit testing
B. Code checkout
C. Docker image build
D. Marketing plan review
Answer: D

6) What is the purpose of caching dependencies in Continuous Integration (CI)?
A. To make builds slower for accuracy
B. To skip testing
C. To speed up repeated builds by reusing unchanged files
D. To reduce code coverage
Answer: C

7) Why is containerization valuable in Continuous Integration/Continuous Delivery (CI/CD) pipelines?
A. Containers ensure consistent environments across stages
B. Containers are slower than virtual machines (VMs)
C. Containers remove the need for testing
D. Containers replace all scripts
Answer: A

8) Which metric is part of the DevOps Research and Assessment (DORA) metrics for DevOps performance?
A. Number of developers on a project
B. Deployment frequency
C. Total code size
D. Meeting hours per week
Answer: B

9) What is a common cause of pipeline failures in Continuous Integration/Continuous Delivery (CI/CD)?
A. Too many reviewers
B. Flaky or unstable tests
C. Fast network connections
D. Use of YAML syntax
Answer: B

10) What is the benefit of ‘fail fast’ in pipeline design?
A. It hides test results
B. It saves time by detecting errors early
C. It ensures all jobs run regardless of failure
D. It only applies to manual testing
Answer: B

11) In a Continuous Delivery (CD) pipeline, what does blue-green deployment help achieve?
A. Simultaneous deployment to all servers
B. Instant rollback and minimal downtime during releases
C. Random user routing
D. Cost reduction by halving infrastructure
Answer: B
