1) Functional requirements describe:
A. System constraints or performance expectations
B. What the system should do
C. The physical network structure
D. Organizational policies only
Answer: B

2) Which of the following is an example of a functional requirement?
A. System shall support 500 users simultaneously.
B. The interface must be intuitive.
C. System shall allow mentors to view student profiles.
D. Response time must be under 3 seconds.
Answer: C

3) Which statement represents a non-functional requirement?
A. Allow mentors to upload a profile picture.
B. Encrypt all user data during transmission.
C. Allow students to log in via university identification (ID).
D. Enable admins to reset passwords.
Answer: B

4) Non-functional requirements are also known as:
A. System constraints
B. Quality attributes
C. Data models
D. Prototypes
Answer: B

5) The “FURPS” (Functionality, Usability, Reliability, Performance, Supportability) model includes all except:
A. Functionality
B. Usability
C. Reliability
D. Productivity
Answer: D

6) Which example is a domain requirement for a university system?
A. Must store mentor contact information.
B. Must integrate with university login system.
C. Should display mentor list within 3 seconds.
D. Must allow password reset feature.
Answer: B

7) Why are domain requirements important?
A. They define visual design rules.
B. They prevent violation of external constraints and regulations.
C. They help reduce code duplication.
D. They determine testing order.
Answer: B

8) Which requirement is poorly written?
A. System shall display mentor list.
B. System shall be user-friendly.
C. System shall validate student identification (ID).
D. System shall log mentor sessions.
Answer: B

9) Functional and non-functional requirements differ because:
A. Functional define features; non-functional define qualities.
B. Both describe coding logic.
C. Non-functional are optional.
D. Functional depend on the hardware.
Answer: A

10) In testing, non-functional requirements are verified by:
A. Functional unit tests
B. Measurement and performance evaluation
C. System walkthroughs only
D. Code review
Answer: B

11) Which type of requirement usually originates from laws or industry regulations?
A. Functional
B. Domain
C. Non-functional
D. Performance
Answer: B

12) Which of the following sets best represents the three types of requirements?
A. Behavioral, System, Logical
B. Functional, Non-functional, Domain
C. Physical, Logical, Conceptual
D. Performance, Usability, Design
Answer: B
